## Page Contracts

### Page 1: `/admin/events/[id]/tournaments/[tid]/launch`

**Purpose:** Pre-launch confirmation screen. Admin reviews roster summary, contextual warnings, phase structure preview, then confirms launch.

**Layout:**
- Breadcrumb: Événements / [Event name] / [Tournament name] / Lancement
- Page heading: "Lancer [Tournament name]"
- Section 1 — Récapitulatif des inscrits: count badge + roster count by check-in status
- Section 2 — Avertissements: zero or more `Alert` components (flowbite-svelte, `color="yellow"`) for contextual warnings
- Section 3 — Structure générée: one collapsible/flat block per phase showing phase name, group count, match count
- Action bar (bottom): "Confirmer le lancement" (primary) + "Annuler" link back to roster page

**States:**
- Loading state: show skeleton while server load fetches roster + phase preview
- No warnings: Section 2 is omitted entirely (not shown as "no warnings")
- Error after failed launch: inline `Alert color="red"` above the confirm button, with "Réessayer" button

**Interaction flow:**
1. Admin arrives on page — sees recap + warnings + preview
2. Admin clicks "Confirmer le lancement" → POST to `/api/tournament/launch`
3. On success → redirect to `/admin/events/[id]/tournaments/[tid]`
4. On failure → show error alert in place, "Réessayer" re-enables the button

### Page 2: `/admin/events/[id]/tournaments/[tid]` (post-launch state)

**Purpose:** Roster page enriched with generated match display after launch.

**Layout additions for `status = 'started'`:**
- Hide the "Lancer" button (tournament already launched)
- Hide check-in controls (launch is locked)
- New section: "Matchs générés" — one subsection per phase
  - **Round-robin / double KO phases:** Table per group — columns: `#` (event_match_id) | Équipe A | vs | Équipe B | Arbitre
  - **Élimination KO phase:** List of matches per round — `Round 1`, `Demi-finales`, `Finale` headings; each row: `#` | Équipe A | vs | Équipe B | Arbitre

**Match table columns:**
| Column | Type | Notes |
|--------|------|-------|
| # | Integer | `event_match_id` — used for fast result entry in Phase 5 |
| Équipe A | Text | Team name(s), "BYE" if null slot |
| vs | Static | Visual separator |
| Équipe B | Text | Team name(s), "BYE" if null slot; "À déterminer" for empty KO bracket slots |
| Arbitre | Text | Referee team name, or "—" if no referee assigned |

### Page 3: Roster page — seeding UI (`is_seeded = true`)

**Purpose:** When `tournament.is_seeded = true`, the roster list gains drag-to-reorder capability for admin to set seed order before launch.

**Visual change:** Drag handle icon (or grip dots) rendered as first column in the roster table rows. Only visible when `is_seeded = true` AND `status` is `ready` or `check-in`.

**Copy:** No new copy — drag behavior is self-evident. Tooltip on grip icon: "Glisser pour réordonner" (optional, consistent with wizard drag-and-drop).

### Wizard additions — PhaseCard (`sets_to_win` / `legs_per_set`)

**Purpose:** Two new fields per phase in the PhasesBuilder wizard step, persisted before launch.

**UI:** Two inline number inputs within the existing `PhaseCard.svelte` component:
- "Sets pour gagner" — integer input, min 1, default 2
- "Jambes par set" — integer input, min 1, default 3

**Validation inline:** `players_per_group` must be a power of 2 (4, 8, 16, 32) for `double_loss_groups` phases — show `Helper` component with `color="red"` if invalid.

## Interaction Contracts

### Confirm button state machine

The "Confirmer le lancement" button follows this state machine:

| State | Button label | Button state | Color |
|-------|-------------|-------------|-------|
| idle | "Confirmer le lancement" | enabled | primary |
| submitting | "Lancement en cours…" | disabled | primary (muted) |
| error | "Réessayer" | enabled | primary |
| success | (redirect — button not shown) | — | — |

### Warning alerts

Each warning is a flowbite-svelte `Alert` with `color="yellow"` and a leading icon.
- Warnings are informational only — they never disable the confirm button.
- Source: CONTEXT.md decision "Pas de condition bloquante — le bouton est toujours actif".

### Cancel launch

Only available to admins with a role above `admin_tournoi` (admin_comite or higher).
Uses the project-standard `confirm()` dialog from `$lib/confirm.svelte.js`.
On confirmation: POST to `/api/tournament/cancel` → redirects to roster page.

### Drag-and-drop seeding

Uses the same drag-and-drop pattern as the wizard's phase ordering (if available in the codebase), or a lightweight HTML5 drag-and-drop implementation.
- Visual: row highlight on drag-over; no animation required.
- Persistence: PATCH to a new `/api/tournament/seed-order` endpoint on drop.
- Only rendered when `tournament.is_seeded = true`.
- Source: CONTEXT.md decision "Drag-and-drop sur la page roster".
