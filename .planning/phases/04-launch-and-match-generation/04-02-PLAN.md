**snake-seeding.ts signature:**

```typescript
export function snakeDistribute(
  teams: string[],
  groupCount: number,
  playersPerGroup: number,
): string[][]
```
Logic per CONTEXT.md: compute `fullGroups = Math.floor(teamCount / playersPerGroup)`. If `fullGroups === 0`, one group with all teams. Apply snake seeding for `fullGroups * playersPerGroup` teams, then append remaining teams to the LAST group. `groupCount` parameter is a hint but actual group count is derived from `Math.floor(teams.length / playersPerGroup)` capped at `groupCount`.

Actually simpler: `groupCount = Math.ceil(teams.length / playersPerGroup)`. Snake for `(groupCount - 1) * playersPerGroup` teams across groupCount groups. Last group gets the remainder.

Wait — re-read CONTEXT.md: "remplir les groupes a players_per_group, le dernier groupe recoit les joueurs restants (groupe plus petit)". So: fill groups at exactly playersPerGroup, and last group gets what remains (could be smaller). So groupCount = Math.ceil(teams.length / playersPerGroup). Snake distribute all teams, first (groupCount-1) groups get exactly playersPerGroup, last gets remainder.

Implementation: standard snake zigzag but stop filling a group once it hits playersPerGroup cap (except last).

**double-ko-group.ts signature:**
```typescript
export function generateDoubleKoGroupMatches(
  group: string[],            // team IDs in seed order
  groupNumber: number,
  phaseId: string,
  startEventMatchId: number,
  config: { setsToWin: number; legsPerSet: number },
): MatchInsertRow[]
```

Per CONTEXT.md, the structure for N players (must be power of 2) is:
- R1: N/2 matches, seeded pairs (S1 vs SN, S2 vs SN-1, ...)
- R2 Upper: N/4 matches among R1 winners
- R2 Lower: N/4 matches among R1 losers, paired to avoid rematch (L-M1 vs L-M(N/2), L-M2 vs L-M(N/2-1))
- R3 Last chance: N/4 matches, R2Upper losers vs R2Lower winners

For 8 players: R1=4, R2U=2, R2L=2, R3=2 = 10 matches total
For 4 players: R1=2, R2U=1, R2L=1, R3=1 = 5 matches total

Generate all match IDs upfront with crypto.randomUUID(). Insert from final round (R3) first, then R2, then R1 — so advances_to_match_id references exist. Actually, since we build in memory and insert all at once in the orchestrator, order does not matter for FK — we just need correct IDs.

R1 matches have team_a_id/team_b_id filled. R2/R3 matches have null teams (filled by Phase 5 when results come in). advances_to_match_id + advances_to_slot wiring must be correct.

**single-elimination.ts signatures:**
```typescript
export function getBracket(participantsCount: number): Array<[number | null, number | null]>

export function generateSingleEliminationBracket(
  teamIds: string[],         // ordered by seed
  phaseId: string,
  startEventMatchId: number,
  tiers: Array<{ round: number; setsToWin: number; legsPerSet: number }>,
): MatchInsertRow[]
```

Use the exact getBracket algorithm from CONTEXT.md. For generateSingleEliminationBracket:
1. Compute bracket size (next power of 2 >= teamIds.length)
2. Use getBracket to get first-round seed pairings
3. Build match tree from root (final) to leaves (first round)
4. Pre-generate all match UUIDs
5. Wire advances_to_match_id from leaf to root
6. Map seed numbers to teamIds (seed > teamIds.length = null = BYE)
7. BYE matches get status 'bye'
8. Apply tiers: tiers[0] = final config, tiers[1] = semi, etc. (round number = tier index from end). If tiers array doesn't cover all rounds, use last tier's config for remaining rounds.

**referee-assignment.ts signature:**
```typescript
export function assignReferees(
  matches: MatchInsertRow[],
  allTeamIds: string[],
  autoReferee: boolean,
): MatchInsertRow[]
```

Returns new array with referee_team_id populated. For each match (processed in event_match_id order), find teams not playing in that match and not already refereeing another match in the same round_number + group_number. Among candidates, pick the one with fewest total referee assignments. If no candidate available, leave null.
