3. Fetch event details:
```sql
SELECT e.id, e.name, e.status, e.starts_at, e.ends_at, e.location, e.registration_opens_at,
       ent.id AS entity_id, ent.name AS entity_name, ent.type AS entity_type
FROM event e
JOIN entity ent ON ent.id = e.entity_id
WHERE e.id = $eventId
  AND e.status != 'draft'
```
4. Fetch tournaments with registration state using LEFT JOINs — compute `is_registered` and `registration_count` in SQL (do NOT load all registrations in TS):
```sql
SELECT
  t.id, t.name, t.category, t.check_in_required,
  COUNT(r.id)::int AS registration_count,
  (r_me.id IS NOT NULL) AS is_registered
FROM tournament t
LEFT JOIN tournament_registration r ON r.tournament_id = t.id
LEFT JOIN tournament_registration r_me
  ON r_me.tournament_id = t.id AND r_me.player_id = $currentPlayerId
WHERE t.event_id = $eventId
GROUP BY t.id, r_me.id
ORDER BY t.name
```
   `$currentPlayerId` = `locals.player?.id ?? null`. When null (no player profile yet), `r_me.id IS NOT NULL` will always be false — no error.
5. Validate tournament rows with `z.array(TournamentWithRegistrationSchema).parse()`.
6. Build and return `EventSchema`-shaped event object + `tournaments: TournamentWithRegistration[]` + `canRegister: event.status === 'ready'`.

**POST (self-register):**
1. `if (!locals.user || !locals.player) error(401, 'Non authentifié')`.
2. Parse body with `z.object({ tournament_id: z.uuid() }).parse(await request.json())`.
3. Verify the tournament belongs to this event AND the event status is `'ready'`:
   ```sql
   SELECT t.id FROM tournament t JOIN event e ON e.id = t.event_id
   WHERE t.id = $tournamentId AND e.id = $eventId AND e.status = 'ready'
   ```
   If 0 rows: `error(404, 'Tournoi introuvable ou fermé')`.
4. Insert registration:
   ```sql
   INSERT INTO tournament_registration (tournament_id, player_id)
   VALUES ($tournamentId, $playerId)
   ```
5. Catch postgres error code `'23505'` (unique constraint) → `error(409, 'Déjà inscrit')`.
6. Return `json({ ok: true })`.

**DELETE (self-unregister):**
1. `if (!locals.user || !locals.player) error(401)`.
2. Parse body same as POST.
3. Delete only if event is still `'ready'` (not started — can't unregister after start):
   ```sql
   DELETE FROM tournament_registration r
   USING tournament t JOIN event e ON e.id = t.event_id
   WHERE r.tournament_id = $tournamentId
     AND r.player_id = $playerId
     AND t.id = $tournamentId
     AND e.status = 'ready'
   ```
4. Return `json({ ok: true })` (idempotent — deleting a non-existent registration is not an error).

Use `sql<Record<string, unknown>[]>` for SELECT queries, `sql` (untyped) for INSERT/DELETE. Import `sql` from `$lib/server/db.js`. Import `json, error` from `@sveltejs/kit`.

**Registration logic (Svelte 5 $state):**
```typescript
let tournaments = $state(data.tournaments)  // local reactive copy

async function register(tournamentId: string) {
  const res = await fetch(`/events/${data.event.id}/register`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ tournament_id: tournamentId })
  })
  if (res.ok) {
    // Optimistic update: flip is_registered, increment count
    const t = tournaments.find(t => t.id === tournamentId)
    if (t) { t.is_registered = true; t.registration_count++ }
  }
}

async function unregister(tournamentId: string) {
  const res = await fetch(`/events/${data.event.id}/register`, {
    method: 'DELETE',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ tournament_id: tournamentId })
  })
  if (res.ok) {
    const t = tournaments.find(t => t.id === tournamentId)
    if (t) { t.is_registered = false; t.registration_count-- }
  }
}
```
