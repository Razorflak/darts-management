**1. Extend TournamentWithRegistrationSchema (lines 176-184):**

Add `start_at` and `partner` as optional/nullable fields so the schema works for both consumers:
- Event page load: SQL selects both — they will be present and coerced
- Dashboard load: SQL does not select them — they will be absent, but the schema accepts that because they are optional

```typescript
export const TournamentWithRegistrationSchema = z.object({
  id: z.uuid(),
  name: z.string(),
  category: CategorySchema,
  check_in_required: z.boolean(),
  registration_count: z.number().int(),
  is_registered: z.boolean(),
  start_at: z.coerce.date().nullable().optional(),  // present on event page, absent on dashboard
  partner: z.string().nullable().optional()          // present on event page, absent on dashboard
})
export type TournamentWithRegistration = z.infer<typeof TournamentWithRegistrationSchema>
```

**2. Add OpenEventSchema after TournamentWithRegistrationSchema:**

The dashboard load builds OpenEvent objects by grouping flat SQL rows. Add the schema so the type can be derived via z.infer<>:

```typescript
export const OpenEventSchema = z.object({
  event: z.object({
    id: z.uuid(),
    name: z.string(),
    status: z.literal("ready"),
    starts_at: z.coerce.date(),
    ends_at: z.coerce.date(),
    location: z.string(),
    entity_name: z.string()
  }),
  tournaments: z.array(TournamentWithRegistrationSchema)
})
export type OpenEvent = z.infer<typeof OpenEventSchema>
```
