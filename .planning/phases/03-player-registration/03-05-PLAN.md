Create all 5 API endpoint files. Each endpoint performs the same admin authorization check before processing.

**Shared admin auth helper pattern** (inline in each file — no separate utility):
```typescript
// Admin auth check (inline in each +server.ts)
if (!locals.user) error(401, 'Non authentifié')
const tournamentRows = await sql<Record<string, unknown>[]>`
  SELECT t.id, e.entity_id FROM tournament t JOIN event e ON e.id = t.event_id
  WHERE t.id = ${params.id}
`
if (tournamentRows.length === 0) error(404, 'Tournoi introuvable')
const entityId = tournamentRows[0].entity_id as string
const roles = await getUserRoles(locals.user.id)
const hasAccess = roles.some(r =>
  r.entityId === entityId &&
  ['adminTournoi', 'adminClub', 'adminComite', 'adminLigue', 'adminFederal'].includes(r.role)
)
if (!hasAccess) error(403, 'Accès refusé')
```

**`register/+server.ts` (POST):**
Uses `z.discriminatedUnion('mode', [...])` schema:
- `mode: 'existing'` + `player_id: z.uuid()`
- `mode: 'new'` + `first_name: z.string().min(1)`, `last_name: z.string().min(1)`, `birth_date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/)`, `licence_no: z.string().optional()`

For `'existing'` mode: use `player_id` directly.
For `'new'` mode: INSERT into player (first_name, last_name, birth_date, licence_no, no user_id) and RETURNING id.

Then INSERT into tournament_registration (tournament_id=params.id, player_id). Catch error code `'23505'` → `error(409, 'Joueur déjà inscrit')`. Return `json({ ok: true })`.

**`unregister/+server.ts` (DELETE):**
Body: `z.object({ player_id: z.uuid() })`.
DELETE from tournament_registration WHERE tournament_id = params.id AND player_id = body.player_id.
Return `json({ ok: true })` (idempotent).

**`checkin/+server.ts` (POST):**
Body: `z.object({ registration_id: z.uuid() })`.
UPDATE tournament_registration SET checked_in = true WHERE id = body.registration_id AND tournament_id = params.id.
Return `json({ ok: true })`.

Also support cancelling check-in: Body with additional `checked_in: z.boolean()` field so a single endpoint handles both check-in and check-out (UPDATE SET checked_in = body.checked_in).

**`checkin-all/+server.ts` (POST):**
No body needed.
Single UPDATE: `UPDATE tournament_registration SET checked_in = true WHERE tournament_id = ${params.id}`.
Return `json({ ok: true })`.

**`players/search/+server.ts` (GET):**
Auth: `if (!locals.user) error(401)` (no role check needed for read-only search).
`const q = url.searchParams.get('q') ?? ''`.
If `q.length < 2`: return `json([])`.
Query with ILIKE on last_name, first_name, licence_no:
```sql
SELECT id, first_name, last_name, birth_date::text, licence_no
FROM player
WHERE last_name  ILIKE ${'%' + q + '%'}
   OR first_name ILIKE ${'%' + q + '%'}
   OR licence_no ILIKE ${'%' + q + '%'}
ORDER BY last_name, first_name
LIMIT 10
```
Validate with `z.array(PlayerSearchResultSchema).parse()`. Return `json(results)`.

Import `getUserRoles` from `$lib/server/authz.js`. Import `sql` from `$lib/server/db.js`. Use `sql<Record<string, unknown>[]>` for all SELECTs.

**API call helpers ($state):**
```typescript
let roster = $state(data.roster)

async function checkIn(registrationId: string, value: boolean) {
  await fetch(`/tournaments/${data.tournament.id}/admin/checkin`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ registration_id: registrationId, checked_in: value })
  })
  const entry = roster.find(e => e.registration_id === registrationId)
  if (entry) entry.checked_in = value
}

async function unregister(playerId: string) {
  await fetch(`/tournaments/${data.tournament.id}/admin/unregister`, {
    method: 'DELETE',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ player_id: playerId })
  })
  roster = roster.filter(e => e.player_id !== playerId)
}

async function checkInAll() {
  await fetch(`/tournaments/${data.tournament.id}/admin/checkin-all`, { method: 'POST' })
  roster = roster.map(e => ({ ...e, checked_in: true }))
}
```
