Réécrire la logique d'insertion des tournois dans `save/+server.ts` pour utiliser la table `phase` au lieu de la colonne `phases JSONB`.

Changements précis :

1. **Ajouter une fonction helper** `insertPhases(tx, tournamentId, phases)` qui :
   - Fait `DELETE FROM phase WHERE tournament_id = tournamentId` (idempotent)
   - Pour chaque phase (index i = position) : INSERT INTO phase avec les bonnes colonnes selon le type

2. **Supprimer la colonne `phases`** des INSERT/UPDATE sur `tournament` :
   - Remplacer `INSERT INTO tournament (event_id, name, club, category, quota, start_time, start_date, auto_referee, phases) VALUES (...)` par `INSERT INTO tournament (event_id, name, club, category, quota, start_time, start_date, auto_referee) VALUES (...)` (sans la colonne phases et la valeur JSON.stringify)
   - Après chaque INSERT tournament, récupérer l'id du tournoi créé (RETURNING id) et appeler `insertPhases(tx, newTournamentId, t.phases)`

3. **Élargir l'ownership check** du UPDATE path :
   - Changer `WHERE id = ${eventId} AND status = 'draft'` → `WHERE id = ${eventId} AND status IN ('draft', 'ready', 'started')`
   - Raison : CONTEXT.md décision — draft, ready, et started sont éditables

4. **UPDATE path** : Le DELETE + re-INSERT des tournois est déjà fait (DELETE FROM tournament WHERE event_id). Comme ON DELETE CASCADE est configuré sur phase(tournament_id), les lignes phase sont automatiquement supprimées quand le tournament est supprimé. Donc pour l'UPDATE path : après chaque INSERT INTO tournament (RETURNING id), appeler insertPhases(tx, newTournamentId, t.phases).

La fonction helper doit distinguer GroupPhase et EliminationPhase :
```typescript
function isGroupPhase(p: Phase): p is GroupPhase {
  return p.type === 'round_robin' || p.type === 'double_loss_groups'
}
```

```sql
  isElim  ? (phase.qualifiers ?? null) : null,
  isElim  ? JSON.stringify(phase.tiers) : null
)
```
