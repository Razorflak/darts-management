**packages/front/src/routes/(app)/events/+page.svelte**

The current formatDate() function:
```typescript
function formatDate(dateStr: string): string {
  if (!dateStr) return '—'
  const d = new Date(dateStr + 'T00:00')
  return d.toLocaleDateString('fr-FR', { day: '2-digit', month: '2-digit', year: 'numeric' })
}
```

After the SQL fix, dateStr will be 'YYYY-MM-DD' or null/undefined. Update formatDate to:
1. Accept `string | null | undefined`
2. Return '' (empty string) for null/missing dates (user expectation: "laisser vide si pas de date saisie")
3. Keep the T00:00 append for local-timezone parsing (correct for YYYY-MM-DD strings)

```typescript
function formatDate(dateStr: string | null | undefined): string {
  if (!dateStr) return ''
  const d = new Date(dateStr + 'T00:00')
  if (isNaN(d.getTime())) return ''
  return d.toLocaleDateString('fr-FR', { day: '2-digit', month: '2-digit', year: 'numeric' })
}
```

Also update the date display in the template to handle empty string gracefully (the separator arrow should not appear when both dates are empty). Current template line:
```svelte
<p class="text-sm text-gray-500">
  {formatDate(event.starts_at)} → {formatDate(event.ends_at)}
</p>
```

Replace with:
```svelte
{#if event.starts_at || event.ends_at}
  <p class="text-sm text-gray-500">
    {formatDate(event.starts_at)}{#if event.starts_at && event.ends_at} → {/if}{formatDate(event.ends_at)}
  </p>
{/if}
```
